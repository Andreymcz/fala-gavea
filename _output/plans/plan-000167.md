# Plan 000167 | perf/backend | 2026-06-24 19:25 UTC | performance: startup warm-up, thread-safe singleton, DB index | Review: light
plan_format_version: 1

## Brief

Fix slow `/reports/query` response on app open: every server restart causes a multi-second cold start because `ChromaSearchClient` (which loads `SentenceTransformer`) is initialized lazily on the first HTTP request instead of at server startup. When 4 parallel requests hit simultaneously on page load, the non-thread-safe singleton can also be initialized multiple times concurrently. A missing index on `reports.created_at` forces a full-table scan + sort on every query.

---

## Why

- **Cold start latency**: `SentenceTransformer` model load takes 2–10 s. Currently deferred to first request → every server restart punishes the first user.
- **Race condition**: `get_report_indexer()` used a plain `if _indexer_instance is None:` check without a lock. In uvicorn's thread pool (sync routes run in threads), 4 simultaneous requests can all pass the check and attempt concurrent model loading — 4× the startup cost and undefined final singleton state.
- **Missing DB index**: `find_page` sorts by `reports.created_at DESC` on every call; no index on that column → full-table scan on each `/reports/query` request regardless of dataset size.
- **4 parallel requests**: frontend mounts `FilterPanel` + `MapView` + `TableView` + `CestaView` (agents) on initial load, each calling `useFilteredReports`. React Query deduplicates same-key queries, but the first 4 fire concurrently before the cache is warm.

---

## Steps

### Step 1 — Thread-safe double-checked locking for ChromaSearchClient singleton
**File**: `src/fala_gavea/presentation/api/dependencies.py`

Add `import threading` and a module-level `_indexer_lock = threading.Lock()`. Replace the bare `if _indexer_instance is None:` guard with double-checked locking:

```python
import threading
_indexer_lock = threading.Lock()

def get_report_indexer() -> IReportIndexer | None:
    global _indexer_instance
    if _indexer_instance is None:
        with _indexer_lock:
            if _indexer_instance is None:
                try:
                    from fala_gavea.infrastructure.chromadb.chroma_search_client import ChromaSearchClient
                    from fala_gavea.infrastructure.embeddings.registry import SemanticConfig
                    _indexer_instance = ChromaSearchClient(SemanticConfig())
                except Exception as exc:
                    _log.warning("ChromaSearchClient unavailable: %s", exc)
    return _indexer_instance
```

The outer check avoids acquiring the lock on the hot path (after initialization). The inner check prevents the race: only one thread initializes the model; the rest find `_indexer_instance` already set when they acquire the lock.

**Already implemented in this conversation.**

### Step 2 — Eager ChromaSearchClient warmup at FastAPI startup
**File**: `src/fala_gavea/presentation/api/main.py`

Register a `@app.on_event("startup")` handler that calls `get_report_indexer()` during server startup, before any HTTP request is served. This moves the cold-start cost from the first user request to server boot time (acceptable for a dev server with `--reload`).

```python
@app.on_event("startup")
def _warm_up_chroma() -> None:
    from fala_gavea.presentation.api.dependencies import get_report_indexer
    get_report_indexer()
```

**Placement**: after `_mount_spa(app)`, before `return app`.

**Already implemented in this conversation.**

### Step 3 — `CREATE INDEX IF NOT EXISTS` for `reports.created_at` at startup
**File**: `src/fala_gavea/infrastructure/database/session.py`

The default query order is `created_at DESC` (used by every `/reports/query` call without semantic ranking). `Base.metadata.create_all()` only creates tables and indexes for *new* databases — adding `index=True` to the column in models.py would not retrofit existing databases. Instead, run an idempotent DDL statement in `create_tables()`:

```python
from sqlalchemy import create_engine, event, text

def create_tables() -> None:
    Base.metadata.create_all(bind=engine)
    with engine.connect() as conn:
        conn.execute(
            text("CREATE INDEX IF NOT EXISTS ix_reports_created_at ON reports (created_at DESC)")
        )
        conn.commit()
```

`CREATE INDEX IF NOT EXISTS` is a no-op when the index already exists; safe to run on every server start.

**Already implemented in this conversation.**
