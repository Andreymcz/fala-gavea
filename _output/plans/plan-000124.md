R1: Force CPU-only PyTorch in Dockerfile; R2: switch `multilingual-e5-base` to `multilingual-e5-small`; R3: unify SentenceTransformer instances into a shared EmbeddingProviderRegistry; R4: replace BERTopic/BERTopicClient with TF-IDF + K-means topics_client.

## Goal

Reduce idle Railway RAM from ~2 GB to ~500–700 MB, dropping estimated cost from ~$22/month to ~$5–10/month. The four changes are:
- R1: Remove CUDA library overhead from Dockerfile (200–600 MB)
- R2: Smaller embedding model (250–400 MB saved; requires ChromaDB re-index after deploy)
- R4: Replace BERTopic + MiniLM with TF-IDF + K-means (300–500 MB saved, no second model)

### Step 4 — Create TfidfTopicsClient replacing BERTopicClient
Docs: N/A

**File:** `src/fala_gavea/infrastructure/topics/tfidf_topics_client.py` (new)

Create a new `TfidfTopicsClient` implementing `ITopicModelPort`. Uses `TfidfVectorizer` + `KMeans` from scikit-learn (already a transitive dep). No embedding model. Fully deterministic with `random_state=42`.

```python
from __future__ import annotations

import logging
from collections import Counter

from fala_gavea.domain.entities.report import Report
from fala_gavea.domain.repositories.semantic_ports import ITopicModelPort

logger = logging.getLogger(__name__)

_DEFAULT_N_CLUSTERS = 8
_DEFAULT_MAX_TERMS = 8


class TfidfTopicsClient(ITopicModelPort):
    """Topic extraction via TF-IDF + K-means. No embedding model required."""

    def __init__(self, n_clusters: int = _DEFAULT_N_CLUSTERS, max_terms: int = _DEFAULT_MAX_TERMS) -> None:
        self._n_clusters = n_clusters
        self._max_terms = max_terms

    def infer_topics(self, reports: list[Report]) -> list[dict]:
        from sklearn.feature_extraction.text import TfidfVectorizer
        from sklearn.cluster import KMeans
        import numpy as np

        texts = [r.text for r in reports]
        if len(texts) < 2:
            return []

        n_clusters = min(self._n_clusters, len(texts))

        try:
            vectorizer = TfidfVectorizer(
                max_features=500,
                ngram_range=(1, 2),
                min_df=1,
                sublinear_tf=True,
            )
            X = vectorizer.fit_transform(texts)
            feature_names = vectorizer.get_feature_names_out()

            kmeans = KMeans(n_clusters=n_clusters, random_state=42, n_init=10)
            labels = kmeans.fit_predict(X)

            counts: Counter[int] = Counter(labels.tolist())
            centers = kmeans.cluster_centers_

            result: list[dict] = []
            for topic_id, count in counts.most_common():
                top_indices = np.argsort(centers[topic_id])[::-1][: self._max_terms]
                terms = [str(feature_names[i]) for i in top_indices]
                result.append({"topic_id": int(topic_id), "terms": terms, "count": int(count)})

            return result

        except Exception as exc:
            logger.error("TF-IDF topic extraction failed: %s", exc)
            return []

    def fit(self, reports: list[Report]) -> None:
        raise NotImplementedError("TfidfTopicsClient does not support pre-fitting.")

    def topic_of(self, report: Report) -> int:
        raise NotImplementedError("TfidfTopicsClient does not support per-document topic lookup.")

    def list_topics(self) -> list[dict]:
        raise NotImplementedError("TfidfTopicsClient does not maintain a persistent topic list.")
```

## Post-deploy steps (manual)

1. **Re-index ChromaDB** — the embedding dimension changes from 768 (`e5-base`) to 384 (`e5-small`). The existing ChromaDB collection will reject queries. Options:
   - Call `POST /admin/seed` with the seed CSV to re-populate (this calls `reindex_all` via `BulkCreateReports`).
   - Or call `DELETE /admin/seed/wipe` first then re-seed.
2. Set env var `FALA_GAVEA_EMBED_MODEL_SEARCH=intfloat/multilingual-e5-small` on Railway (optional if Dockerfile default is sufficient).
