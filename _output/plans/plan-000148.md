## Brief

Populate seed of encaminhamentos based on existing API URL. Filter relatos and build a 50% of relatos encaminhados.

## Context

The project already has three seed scripts in `scripts/`:
- `seed_users.py` — inserts dev users (admin/citizen01/agente) via API + direct DB promotion
- `seed_report_types.py` — creates 8 initial ReportTypes via API
- `seed_relatos.py` — bulk-inserts reports via API using CSV templates

No seed exists for `Forwarding`. The `POST /forwardings` endpoint requires `agent`+`admin` role. `CreateForwarding` atomically links N reports and transitions their status to `encaminhado`. The goal is a script that:
- Authenticates as the dev agent (`agente@gavea.br`)
- Fetches all `pendente` (and optionally `em_analise`) reports via `POST /reports/query` (paginating at limit=200)
- Takes a 50% random sample of those reports
- Groups the sample by `report_type_id`, then splits large groups into sub-batches of ≤ `--batch-size` reports
- Maps each report_type name to a realistic institution and proposed solution
- POSTs one `Forwarding` per sub-batch
- Reports counts: created / skipped / errors

## Design Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Auth user | `agente@gavea.br` / `agente12345` | Only agent+admin can POST /forwardings; agent is the natural actor |
| Status filter for candidate reports | `["pendente"]` by default, `--include-em-analise` flag to add `em_analise` | Avoid re-encaminhating already-encaminhado/resolvido reports; `pendente` is the natural source for new forwardings |
| 50% sample | Shuffle full candidate list, take `ceil(total * 0.5)` | Deterministic given `--seed` flag for reproducibility |
| Grouping | By `report_type_id` | Creates semantically coherent forwardings; mirrors how a real agent would batch similar problems |
| Sub-batch size | Max 20 per forwarding (configurable via `--batch-size`) | Realistic forwarding cardinality; avoids hitting API limits |
| Institution mapping | Static dict keyed on report_type name | The 8 seed report_types are stable; mapping keeps script self-contained |
| Idempotency | Skip if `GET /forwardings` returns > 0 entries, unless `--force` | Mirrors pattern in seed_relatos.py |
| HTTP client | `httpx` (sync) | Already a project dependency; consistent with seed_relatos.py style |

## Steps

### Step 1: Create scripts/seed_forwardings.py

Create a standalone seed script that authenticates as the dev agent, queries all `pendente` reports via `POST /reports/query` (paginating until exhausted), draws a 50% random sample, groups by `report_type_id` into sub-batches of ≤ `--batch-size`, and POSTs one forwarding per sub-batch via `POST /forwardings`.

**Script interface:**
```
uv run python scripts/seed_forwardings.py [OPTIONS]

Options:
  --url URL              Base API URL (default: http://localhost:8000)
  --user EMAIL           Agent login email (default: agente@gavea.br)
  --password PWD         Agent password (default: agente12345)
  --batch-size N         Max reports per forwarding (default: 20)
  --include-em-analise   Also include em_analise reports as candidates
  --seed N               Random seed for reproducibility (default: 42)
  --force                Create forwardings even if some already exist
```

**Implementation outline:**

```python
1. parse args
2. login → get token (POST /auth/token, form-encoded)
3. idempotency check → GET /forwardings (agent auth); if count > 0 and not --force, print warning and exit 0
4. fetch all report_types → GET /report_types → build {id: name} dict
5. paginate POST /reports/query with statuses filter, limit=200, offset=0..N
   → collect all report items (id + report_type_id)
6. shuffle with random.seed(seed); take [:ceil(len * 0.5)]
7. group by report_type_id → dict[type_id, list[report_id]]
8. for each type_id, split into sub-batches of batch_size
9. for each sub-batch: POST /forwardings {institution, proposed_solution, report_ids}
10. print summary: created / errors
```

The institution and proposed_solution are derived from the report_type name using the static mapping in the Design Decisions table above. If the report_type name is not in the map, use the default `("Subprefeitura da Gavea", "Analise tecnica e encaminhamento ao orgao competente")`.
