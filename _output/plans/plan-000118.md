# Plan 000118 | fix/seed | 2026-06-20 23:00 UTC | fix seed duplicate email race | Review: light
plan_format_version: 1

## Brief

ERROR: UNIQUE constraint failed on users.email during database seeding.

```
sqlalchemy.exc.IntegrityError: (sqlite3.IntegrityError) UNIQUE constraint failed: users.email
[SQL: INSERT INTO users (id, email, password_hash, name, role, created_at) VALUES (?, ?, ?, ?, ?, ?)]
[parameters: ('d1272a3a-450f-42f0-920c-2db0c91fbe7b', 'c0002@seed.gavea.br', ...)]
```

## Root Cause

`BulkCreateReports.execute()` uses a `find_by_email → save` pattern that is not atomic.
Two concurrent `POST /seed/relatos` requests (or a second upload after a partial seed on Railway restarts) can both call `find_by_email(email) → None`, both call `User.create()` (generating distinct UUIDs), then both call `user_repo.save()`. The `save()` method guards by primary key (`session.get(UserModel, user.id)`), not by email — so both attempts reach `INSERT` and the second one hits the `UNIQUE` constraint on `users.email`, raising an unhandled `IntegrityError` that crashes the entire request (no rows from that point are inserted).

The fix belongs in the infrastructure layer (`SQLAlchemyUserRepository.save()`) so the use-case code stays clean: catch `IntegrityError` after `commit()`, rollback the session, and re-fetch the existing user by email.

## Steps

### Step 1: Make `SQLAlchemyUserRepository.save()` idempotent on duplicate email

In `src/fala_gavea/infrastructure/repositories/sqlalchemy_user_repository.py`, wrap the `commit()` call in a try/except for `sqlalchemy.exc.IntegrityError`. On conflict, rollback and re-fetch the existing user by email. This makes the seeding endpoint idempotent across concurrent or repeated calls without touching the use-case layer.

Concrete change to `save()`:

```python
from sqlalchemy.exc import IntegrityError

def save(self, user: User) -> User:
    model = self._session.get(UserModel, user.id)
    if model is None:
        model = self._to_model(user)
        self._session.add(model)
    else:
        model.email = user.email
        model.password_hash = user.password_hash
        model.name = user.name
        model.role = user.role.value
    try:
        self._session.commit()
    except IntegrityError:
        self._session.rollback()
        existing = self._session.scalars(
            select(UserModel).where(UserModel.email == user.email)
        ).first()
        if existing:
            return self._to_entity(existing)
        raise
    self._session.refresh(model)
    return self._to_entity(model)
```
